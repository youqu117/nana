# 产品规划（中文版）

> 文档目的：给开发、设计、测试统一版本目标与迭代优先级。
> 当前基线版本：`1.1.0`

## 1. 产品定位

Nana 是一款轻陪伴型像素桌宠应用，核心目标是：

1. 长驻可见：桌宠在系统层悬浮，低打扰地陪伴用户。
2. 可互动：点按、拖拽、小屋互动形成即时反馈。
3. 可成长：状态与等级系统驱动长期留存。
4. 可扩展：资源包驱动新宠物、新动作、新互动。

## 2. 当前版本能力（1.1.0）

- 桌宠悬浮服务与多实例管理
- 领养、喂食、重命名、召唤/回家
- 小屋互动（音乐、睡觉、问候、唱歌）
- 宠物等级与进度展示
- FocusLock + Todo 防沉迷流程
- 自定义素材导入（ZIP / 图片）

## 3. 近期迭代目标（1.1.x）

### 3.1 互动增强

- 新增更多自主行为（发呆、巡视、随机表情）
- 互动强度与等级、情绪状态联动更细化
- 宠物差异化行为模板（如龙系火焰、qiqi 音乐节奏）

### 3.2 视觉与动画

- 统一特效素材风格（音符、表情、粒子）
- 完善动作连续性，减少“僵硬感”
- 增加更多可复用 FX 资产与触发曲线

### 3.3 结构与质量

- 继续清理历史编码与文案乱码
- 提升资源命名规范与文档一致性
- 完成关键路径静态回归（编译 + lint 核心错误）

## 4. 中期规划（1.2.x）

- 多宠同屏策略优化（性能与交互冲突处理）
- 互动脚本配置化（JSON 驱动行为编排）
- 个性化主题包（背景、UI 色板、特效包）

## 5. 长期方向（2.x）

- 跨设备云同步（宠物成长数据）
- 更丰富的成长树与解锁机制
- 社区化资源分享（审核后的内容包）

## 6. 技术侧优先级

1. 稳定性优先：确保编译与核心流程可回归。
2. 架构一致性：模块职责单一、包结构清晰。
3. 可维护性：文档与代码同步更新。
4. 可扩展性：互动事件与资源结构保持配置化。

## 7. 发布前检查清单

- `:app:compileDebugKotlin` 通过
- Manifest、布局自定义类路径一致
- 小屋互动主流程可用
- FocusLock 主流程可用
- 文档（README/USAGE/STRUCTURE）同步
