# 像素桌面桌宠 App 方案（PRD + 交互规格 + 资产规范 + 技术落地）

> 目标：能做出来、能扩展、后期可商业化但不绑死。默认 Android 完整体验，iOS 给出可落地的受限实现策略。

---

# 0. 项目概述

## 0.1 产品一句话

一款常驻手机屏幕边缘的像素桌宠（猫/狗为主），支持领养、命名、年龄成长、情绪变化与触摸互动；桌宠在桌面与任意 App 上层自由移动，提供轻量陪伴与收集换装乐趣。

## 0.2 目标与边界

1. 核心目标
   (1) 桌宠“始终可见、可交互、轻打扰”
   (2) 点击即反馈（转圈、吐舌头、卖萌表情）
   (3) 领养后自动计时成长（年龄、阶段、解锁动作/外观）
   (4) 可扩展内容：更多宠物、动作包、配饰、主题

2. 非目标（第一版不做或弱化）
   (1) 复杂战斗/社交系统
   (2) 深度剧情、多结局养成
   (3) 需要大量文本剧情的重内容
   (4) 强提醒、高频通知骚扰

# 1. 平台策略（必须写清）

## 1.1 Android（主目标：完整形态）

1. 形态：系统悬浮窗（全局覆盖层），可显示在桌面与其他 App 上层
2. 关键权限：悬浮窗权限（SYSTEM_ALERT_WINDOW）；可选前台服务（Foreground Service）保证稳定；可选忽略电池优化（提升驻留稳定）
3. 交互：悬浮宠物可拖拽、吸附边缘、自动走动、点击触发动作

## 1.2 iOS（现实约束：无法等价实现“跨 App 悬浮”）

iOS 通常不允许第三方 App 常驻覆盖在其他 App 上层。可落地的折中组合：

1. 桌面小组件（Widget）：展示桌宠静态/轻动画（依系统限制）
2. 应用内“悬浮桌宠”：只在本 App 内实现全局漂浮（可做得和 Android 一样）
3. Live Activities / 动态岛（若适配机型）：显示状态条信息（心情、饥饿、年龄）
4. 结论：iOS 体验目标定为“桌面可见 + 应用内可玩 + 状态可见”，但不承诺跨 App 悬浮

# 2. 用户故事与体验原则

## 2.1 用户故事（核心）

1. 我领养一只像素猫/狗，给它取名，从此开始计时成长
2. 我在桌面/刷别的 App 时，它在角落活动，不挡内容
3. 我点一下，它会吐舌头/眨眼/摇尾巴；再点会转圈跑
4. 我拖拽它到屏幕另一侧，它会落地后抖一抖继续走
5. 我打开面板能看到年龄、精力、心情、亲密度，并调整大小透明度与勿扰时段

## 2.2 体验原则（落地执行用）

1. 弱打扰：默认靠边、自动避开触控热点区域（键盘、底部导航）
2. 强反馈：任何点击 150ms 内必须有视觉反馈（表情/粒子/动作启动）
3. 低学习成本：只需要“点、拖、双击、长按”四类操作
4. 低电耗：后台降帧、静止降频；屏幕熄灭停止渲染
5. 可持续扩展：宠物=配置+素材包，动作=可插拔资源，不为每个宠物写死逻辑

# 3. 功能范围（MVP 必做 + 扩展可做）

## 3.1 MVP 功能清单（第一版上线即可）

1. 领养与账号
   (1) 游客模式可玩；登录可同步（手机号/邮箱/第三方任选）
   (2) 领养 1 只宠物，命名，记录领养时间
   (3) 展示年龄（天/周/月）与成长阶段

2. 悬浮桌宠（Android 完整）
   (1) 可拖拽移动、边缘吸附
   (2) 自动随机走动（屏幕边缘区域内巡航）
   (3) 点击互动动作（至少 3 个：吐舌头、转圈、眨眼/摇尾巴）
   (4) 简单碰撞边界与回弹
   (5) 一键隐藏/显示；勿扰模式（定时或手动）

3. 情绪与状态（轻量三指标）
   (1) 精力 Energy：影响走动频率与睡觉
   (2) 心情 Mood：影响表情与动作池权重
   (3) 亲密 Affection：影响卖萌概率、靠近用户点击点的倾向

4. 抽屉面板（管理入口）
   (1) 宠物档案：名字、年龄、阶段、三指标条
   (2) 设置：大小、透明度、吸附强度、移动频率、勿扰时段
   (3) 动作预览：点击触发某动作（测试/娱乐）
   (4) 资源：切换到预设猫/狗模型（至少 2 个宠物）

## 3.2 V2/V3 扩展（不影响架构）

1. 多宠物同屏（最多 2-3 只，默认 1）
2. 换装系统（项圈/帽子/眼镜/背包）
3. 动作包解锁（成长或任务）
4. 轻任务：每日抚摸 10 次、喂食 1 次、玩耍 1 次
5. 主题粒子：爱心泡泡、落叶、雪花（可关）

# 4. 核心交互规格（手势与反馈）

## 4.1 手势定义（全局一致）

1. 单击 Tap

* 触发：卖萌动作池（吐舌头/眨眼/摇尾巴）随机 1 个
* 冷却：0.8s（防狂点破坏节奏）

2. 双击 Double Tap

* 触发：强动作（转圈 Spin 或短跑 Run）
* 冷却：2.0s

3. 长按 Long Press（≥ 350ms）

* 进入 Held：被抓起（显示抓起特效/阴影）
* 手指移动：宠物跟随指尖
* 松开：落地（落地帧 + 抖动回弹），回到 Idle 或 Walk

4. 拖拽 Drag（在长按后）

* 可吸附屏幕边缘，吸附后进入 Docked（停靠待机）

## 4.2 不打扰策略（默认开启）

1. 屏幕底部 15% 区域设为“低进入区”（避免遮挡系统导航/输入法）
2. 键盘弹出时：自动上移到键盘上方或进入边缘 Docked
3. 全屏游戏/视频：可选“自动隐藏”或“降透明+静止”

# 5. 桌宠系统：资产规范（像素分层 + 可交互前提）

你提的“区分四肢和头身体舌头”要写成硬规范，否则后续交互会崩。

## 5.1 统一像素规格

1. 基准像素网格：1 px 不做缩放模糊，渲染只允许整数倍缩放（2x/3x/4x）
2. 推荐部件贴图尺寸：48×48（更细腻）或 32×32（更复古）
3. 动画帧率：默认 12 fps（动作类），待机可 6 fps（省电）
4. 输出：PNG（透明）+ JSON（帧序列/锚点/碰撞盒）

## 5.2 必备分层（最小集合）

1. body（身体）
2. head（头）
3. face（表情层：眼/眉/嘴；可拆成 eyes、mouth 更灵活）
4. leg_fl / leg_fr（前肢左右）
5. leg_bl / leg_br（后肢左右）
6. tail（尾巴）
7. tongue（舌头）
8. acc_1 / acc_2（配饰层，可空）

## 5.3 锚点与碰撞盒（强制）

1. anchor（挂点）

* 每个部件必须有相对 body 的锚点坐标，如 head_anchor=(24,14)
* face 锚点相对 head，如 face_anchor=(24,22)

2. hitbox（点击/拖拽盒）

* 定义一个矩形或多边形区域，优先矩形即可
* hitbox 不随透明像素变化，避免“点不到/误触”

3. bounds（运动边界）

* 每帧可计算当前整体包围盒，用于屏幕边缘碰撞

# 6. 行为与状态机（让互动“可配置、可扩展”）

## 6.1 状态机总览

主状态（示例）：
Idle、Walk、Run、Spin、Cute、Sleep、Held、Docked、Hidden

## 6.2 状态优先级（从高到低）

1. Hidden（隐藏）
2. Held（被抓起拖动）
3. Spin/Run/Cute（强制动作）
4. Sleep（睡眠）
5. Walk（随机走动）
6. Docked（停靠待机）
7. Idle（默认待机）

## 6.3 触发规则表（可直接写进配置）

| 触发         | 条件          | 进入状态       | 时长               | 结束回到      |
| ---------- | ----------- | ---------- | ---------------- | --------- |
| Tap        | cooldown_ok | Cute(随机)   | 0.6-1.2s         | Idle/Walk |
| DoubleTap  | cooldown_ok | Spin 或 Run | 0.8-1.5s         | Idle      |
| LongPress  | ≥350ms      | Held       | 持续               | 松手后 Idle  |
| Inactivity | 30-90s 无交互  | Sleep      | 直到 Energy≥阈值或被点醒 | Idle      |
| TimerTick  | 每 N 秒       | Walk(随机)   | 1-4s             | Idle      |
| Dock       | 靠边吸附        | Docked     | 直到被拖/点           | Idle/Cute |
| LowEnergy  | Energy < 15 | Sleep 权重↑  | -                | -         |

# 7. 情绪与数值系统（轻量但“看起来像活的”）

## 7.1 数值定义（0-100 或 -100~100）

1. Energy：0~100（精力）
2. Affection：0~100（亲密度）
3. Mood：-100~100（心情）

## 7.2 每日与每小时变化（建议默认值）

1. Energy

* 每 10 分钟自然下降 1（屏幕亮且可见时）
* 睡觉状态每 10 分钟恢复 3
* 被频繁打扰（连续拖拽 3 次）短暂下降 2

2. Affection

* 每次有效 Tap +1（0.8s 冷却，仅计一次）
* 每日上限 +30（防刷）
* 长期不互动（24h）缓慢衰减 3

3. Mood

* 初始 0
* Energy 高于 60：Mood 每 30 分钟 +1（上限 +40）
* Energy 低于 20：Mood 每 30 分钟 -2（下限 -60）
* Tap/玩耍：Mood +2
* 被狂拖：Mood -3（增加“有脾气”的真实感）

## 7.3 情绪到表现的映射（直接可用）

| Mood 区间    | 表情      | 动作权重倾向                     |
| ---------- | ------- | -------------------------- |
| 40 ~ 100   | 开心脸/星星眼 | Walk/Run/Spin 权重↑，Cute 更频繁 |
| -20 ~ 40   | 普通脸     | 正常 Idle/Walk               |
| -60 ~ -20  | 低落脸     | 趴着/慢走/Sleep 权重↑            |
| -100 ~ -60 | 生气/委屈   | 跺脚/炸毛(猫)/低吼(狗) 权重↑，Cute 变少 |

# 8. 成长与年龄（领养即开始计时）

## 8.1 时间与阶段

1. 领养时间：AdoptAt（UTC 存储，本地显示）
2. 年龄显示：

* < 7 天：显示 X 天
* 7-28 天：显示 X 周
* ≥ 28 天：显示 X 月（可保留天数细分）

3. 阶段划分（建议）

* 幼崽：0-3 天（动作少、卖萌多）
* 少年：3-14 天（解锁跑、转圈更频繁）
* 成年：14 天+（解锁换装槽/主题粒子）
* 老年：90 天+（可选，仅表情/动作微调，不要做负反馈惩罚）

## 8.2 解锁机制（低成本高反馈）

1. 每阶段解锁

* 1 个新动作
* 1 个新表情
* 1 个装扮槽位或配色

2. 解锁展示

* 解锁时弹出小卡片 + 立即播放新动作预览 1 次

# 11. 悬浮引擎（Android 落地关键点）

## 11.1 渲染与性能

1. 帧率策略

* 交互/移动中：30 fps
* 静止 idle：12 fps
* 后台或屏幕熄灭：停止渲染与 tick（只记录时间）

2. Tick 逻辑

* 每 1s 做一次轻量逻辑更新（数值、状态权重）
* 每帧只做动画帧推进与位置插值，避免重计算

3. 电量策略

* 使用前台服务时展示最小化通知（Android 系统要求）
* 检测电池优化限制时提示用户设置白名单（可选）

## 11.2 移动算法（简单但自然）

1. 屏幕可活动区域：屏幕宽高扣除安全边距（顶部状态栏、底部导航/手势区）
2. 行为：

* 随机目标点：在边缘带内采样（例如距边缘 0~120px）
* 使用缓动：ease-in-out，避免直线机械感
* 到达后停顿 0.5~2s 再选新目标

3. 边缘吸附

* 靠近边缘 < 24px 自动吸附
* 吸附后进入 Docked，Idle 动画播放，但位置固定

## 11.3 点击与拖拽判定（防误触）

1. Tap 只在“按下-抬起位移 < 8px 且时间 < 250ms”判定
2. LongPress ≥ 350ms 直接转 Held，取消 Tap
3. Drag 只允许在 Held 状态发生（体验一致）

# 12. 版权与合规（别后期踩坑）

1. 悬浮权限说明：首次开启需明确用途与可关闭方式
2. 隐私：默认不采集屏幕内容，不做无关权限（通讯录、短信等一律不需要）
3. 广告/付费：若后期加入，保证不影响基础互动体验（不把“抚摸”变成广告按钮）

# 13. QA 验收标准（上线前可用）

## 13.1 功能验收

1. 悬浮窗在桌面与任意 App 均可显示（Android）
2. 4 手势稳定：Tap/DoubleTap/LongPress/Held 拖拽不冲突
3. 1 分钟无操作进入 Sleep；点醒恢复
4. 年龄计时正确：杀进程、重启手机后仍正确
5. 设置项生效：透明度/大小/勿扰/隐藏立即生效

## 13.2 性能验收（建议指标）

1. 常驻 30 分钟 CPU 平均占用低（目标 < 3-5% 视机型）
2. 内存稳定，不随动画播放持续增长
3. 不出现明显掉帧与触控延迟

# 附录 A：anim.json 字段规范（Sprite Sheet 动画 + 锚点 + 事件点）

## A1. 设计目标

只靠 JSON 就能定义：帧序列、每帧裁剪、播放 fps、循环、翻转、偏移、锚点、碰撞盒。

事件点可用于：吐舌头时播放“爱心粒子”、转圈第 3 帧发出“SpinStart”。

同一规范可用于：整图动画（MVP）与分层动画（后续升级）。

## A2. anim.json 顶层结构（Schema）

```json
{
  "version": 1,
  "id": "dog_shiba_spin",
  "type": "spritesheet",
  "image": "sheet.png",
  "pixel_art": true,
  "fps": 12,
  "loop": false,
  "duration_ms": 1000,
  "origin": { "x": 0, "y": 0 },
  "scale_hint": 3,
  "anchors": {
    "root": { "x": 24, "y": 40 },
    "head": { "x": 24, "y": 16 },
    "face": { "x": 24, "y": 22 },
    "tail": { "x": 10, "y": 28 },
    "tongue": { "x": 30, "y": 26 }
  },
  "hitboxes": {
    "tap": { "shape": "rect", "x": 8, "y": 10, "w": 32, "h": 30 },
    "body": { "shape": "rect", "x": 10, "y": 14, "w": 28, "h": 26 }
  },
  "frames": [
    {
      "i": 0,
      "rect": { "x": 0, "y": 0, "w": 48, "h": 48 },
      "offset": { "x": 0, "y": 0 },
      "duration_ms": 83,
      "events": []
    }
  ],
  "timeline": [
    { "t_ms": 0, "event": "ANIM_START" },
    { "t_ms": 250, "event": "SPIN_PEAK" }
  ],
  "tags": ["spin", "cute"],
  "notes": "Spin animation for shiba dog."
}
```

## A3. 字段逐项说明（建议严格按这个实现）

### 基本元信息

- version：整数，便于未来升级兼容。
- id：动画唯一 ID（建议 petId_action）。
- type：固定 "spritesheet"（后续可扩展 "layered"）。
- image：sprite sheet 图片文件名（相对 anim.json 所在目录）。
- pixel_art：true 表示渲染必须整数倍缩放、禁用双线性滤波。

### 播放控制

- fps：默认帧率，播放器可据此计算每帧时长。
- loop：是否循环（idle/walk/sleep 一般 true；spin/cute 一般 false）。
- duration_ms：可选；若给出则播放器以此为准对帧时长做均匀/比例缩放。
  - 优先级建议：frames[].duration_ms > duration_ms > fps。

### 坐标与缩放提示

- origin：渲染原点（一般 0,0）；后期如果做 atlas 合并可用。
- scale_hint：建议缩放倍数（例如 3x）；不是强制，只是默认 UI 值。

### 锚点 anchors（为未来“分层/挂件/表情层”做准备）

- anchors.root：宠物脚底或重心位置（用于贴地、吸附边缘、抖动）。
- head/face/tail/tongue：部件挂点，若当前动画是整图也保留（方便对齐粒子/表情）。
- 锚点坐标的约定：以“单帧裁剪矩形 rect”左上角为 (0,0)，向右 x 正，向下 y 正。

### hitboxes（点击与碰撞盒）

- tap：点击判定区域（推荐固定，不随帧变，减少误触）。
- body：可选，用于碰撞或拖拽抓取更精确。
- 支持 shape：rect（先实现它就行）。

### frames（帧序列）

每一帧对象字段：

- i：帧序号（0-based，可选但建议有）。
- rect：从 sprite sheet 裁剪区域。
- offset：渲染偏移（用于转圈时画面居中不抖）。
- duration_ms：该帧显示时长（可选；不写则用 fps 推导）。
- events：该帧触发的事件数组（帧事件点）。

### timeline（按时间触发事件，可选）

如果你不想把事件绑到某个帧，可以用时间轴：

- t_ms：时间点（相对动画开始）。
- event：事件名。

实现上你可以先只实现 frames[].events，timeline 以后再加。

### tags/notes（可选）

- tags：用于动作分类（cute/idle/move）。
- notes：给美术/策划看的说明。

## A4. 推荐事件命名（直接可用）

事件名建议全大写、下划线：

- ANIM_START、ANIM_END（播放器可自动发，也可在 timeline 里写）。
- EMIT_HEART（发射爱心粒子）。
- TONGUE_OUT / TONGUE_IN（吐舌头伸出/收回，若你要做分层舌头可用）。
- FOOTSTEP（走路脚步声/震动）。
- SPIN_PEAK（转圈峰值，可能触发眩晕星星粒子）。
