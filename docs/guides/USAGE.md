# 使用说明（中文版，v1.1.0）

适用版本：`app/build.gradle` 中 `versionName = 1.1.0`

## 1. 功能概览

- 首页：启动/停止桌宠悬浮服务，查看当前状态。
- 我的宠物：查看、喂食、重命名、召回/召唤、删除。
- 商店：从内置资产领养宠物实例。
- 小屋：进行音乐、睡觉、问候、唱歌等互动。
- 设置：缩放、透明度、漂移、导入资源管理。
- FocusLock：专注模式 + Todo 防沉迷流程。

## 2. 首次使用步骤

1. 安装并打开 App。
2. 授予悬浮窗权限（桌宠显示必需）。
3. 到“商店”领养至少 1 只宠物。
4. 回首页开启桌宠服务。
5. 回到桌面或其他应用观察桌宠悬浮运行。

## 3. 基础交互

- 单击宠物：提升心情/亲密并触发反馈。
- 双击宠物：触发强动作（如 Run/Spin）。
- 长按拖拽：移动桌宠位置。
- 小屋互动：可触发音符、睡眠、问候、唱歌、喷火（龙）等效果。

## 4. FocusLock（防沉迷）

1. 进入 FocusLock 页面。
2. 开启无障碍服务（按页面提示）。
3. 选择或创建 Todo 任务。
4. 启动专注后，阻断名单应用会被锁定页拦截。

## 5. 自定义素材导入

### 导入 ZIP

1. 打开“设置”页面。
2. 选择“导入 ZIP”。
3. ZIP 中应包含可识别的 `manifest.json`（或可生成最小清单的图片结构）。
4. 导入完成后会自动重扫资产。

### 导入图片

1. 设置页选择“导入图片”。
2. 输入宠物名称。
3. 系统将生成最小可运行清单并加入资产列表。

## 6. 常见问题

### Q1：桌宠不显示

- 先确认悬浮窗权限是否开启。
- 检查首页开关是否为“运行中”。
- 确认至少有一只宠物被领养并启用。

### Q2：FocusLock 不生效

- 确认无障碍服务已开启。
- 检查是否已启动专注状态。
- 确认阻断名单或 Todo 已正确设置。

### Q3：互动特效没有出现

- 先进入小屋页面。
- 确认宠物已成功加载。
- 低性能机型可能出现动画延迟，但不影响状态变更。

## 7. 维护建议

- 变更目录结构后，务必同步更新文档与 Manifest。
- 自定义素材建议先放测试包再批量导入。
- 重要版本发布前至少执行：

```powershell
.\gradlew.bat :app:compileDebugKotlin
```
